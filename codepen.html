CodePen

Introduction

CodePen is an online platform for people who want to code, whether it is a developer, student, teacher, or expert. CodePen uses front-end languages like HTML, CSS and JavaScript. Users can create "Pens" where they can code, and CodePen will render the code live in the browser window. Users can also interact with other users and follow them, which also offers an outstanding way to get new inspirations for coders. The CodePen users are front-end developers. The editor allows users to skip syntax and write main content. Sometimes this also leads to confusion among new coders as they're not exceptionally skilled. I think to use the platform initially, intermediary practice in Web Development is essential.

Community & Support

Web Development is well established and thriving. The gigantic online communities are available on the internet use CodePen. So, if you happen to hit a wall with your code, the online community will be saving your day (even on Slack). Also, technical support is exceptional; when I was planning to become a PRO user, it cost me a little extra. The pricing was in USD, not in CAD. After contacting support, they returned the amount under 24 hrs.

The community also helps each other in:
· Sparks: a collection of Pens and Posts from the users.
· Challenges: allows to work on and level up skills.
· Topics: can make a user learn new languages/frameworks from scratch with illustrations.

For Education

First-time users are the vertebrae, so there's an extreme focus to keep them sticking. But, how does it feels for the students and instructors to be first-time users? Fortunately, I've been both, and there is one account for all. There are various modes that users can toggle in-between pens, whether that user is a student or an instructor. After simply signing-up:
· Students can click on "Pen" and code, or the instructor can add them to the class via a link. Students can also chat during class. Students can use "Collab Mode" with other users to engage in more learning experiences.
· Instructors can use "Professor Mode" at any point while coding to allow students to join. They can collaborate with students or use "Presentation Mode" if they teach in class on a projector (we did that in class).
The most crucial part of the learning is getting feedback. Students get live feedback on coding by getting output in real-time. Another cool thing that I like is that students can share their pen's link instead of sending the whole GitHub repository. CodePen allows outstanding feedback as it enables emmets and snippets for faster coding.

Challenges

Challenges that I, as a user, experienced:
· To use features that involve more interaction, one must be a PRO user. Which could be a little heavy on the pocket. Having a free account will only do the necessary things.
· In professor mode, only the chat function is available. No voice or video chat support. But, hey, we're here to watch the code, Right? It's not meeting software. But at least, voice chat would've helped.

My Experience

It's helping the students to learn code without feeling the need for a robust machine, which is also cost-effective. Students don't have to ask for code from the instructors. The code is there at all times, saved in pen. The learning curve even gets higher when entering into collab mode; students can learn new skills by collaboration. Comforting themes and graphics interact with the user to prevent boredom. The rock-solid thing that I found was this platform's engagement; students can build using the latest technologies (React, Flutter etc.). Also, using various frameworks, libraries, and UIs to learn something entirely. Finally, the question arises with all these things in hand, does it maintain student's interest? The answer is YES!

Conclusion

I believe in learning by going and breaking things. In my experience, it's a potent tool for CS Education. An excellent way to see clunky mistakes and learning from them. With PRO, there is a whole sea of inspiration to dive in. I think one must use it if someone is into web dev if they haven't already. In few words, it is like a must-have item on your grocery list.
Happy Coding.
